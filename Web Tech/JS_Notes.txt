Scope
    Scope determines where a variable can be accessed in our code 
    Types
        Block Scope 
            The area inside the {} -> block,
             the variables which are delclared inside can't be access it from outside of the scope except var

        Local / Functional scope
            The area inside the Function block, it belongs to that particular Function we can't access it from outside
        
        Global scope
            The area outside the Functions and any block are called global scope
        
        Lexical scope
            the inner Function can access to the variables of its parent scope

Closure
        The inner Function will remember the outer Function variables even after the outer Function executed

variable Declaration
    Types   
        var
            Redeclaration & Reinitialization is posiible
            while declaring initialization is not mandatory
            variables will hoisted and initialixed as undefined
            It follows global scope
        Let
            Redeclaration is not possible while Reinitialization is possible
            while declaring initialization is not mandatory
            variable will hoisted but remains uninitialized in Temprory dead zone
            It follows Function scope
        Const
            Redeclaration and Reinitialization is not possible
            while declaring initialization is mandatory
            variable will hoised but remains uninitialized in tempropry dead zone
            It follows Function scope
        
Hoisting
    The process of moving the declaration part to the top of the scope

TDZ
    Tempropry dead zone, It is the period of time btw declaration and initialization

Function
    It is the set of instruction/ block of code, which is used to perform a particular task
    Types
        Named Function
            A Function which having name is called as named Function
                function add(a,b){
                    return a+b;
                }

        Anonymos Function
            A Function without name is called as Anonymos Function
                function (a,b){
                    return a*b;
                }

        Function Expression
            A Function delclared inside the variable is called Function Expression
                const add = named/ Anonymos Function

        Arrow Function
            A function which is created using arrow insted of function keyword is called as arrow function
                let add = (a,b)=>{
                    return a+b;
                }

        IIFE (Immediate invoked Function Expression)
            This Function will executed suddenly at once ofter its declaration and it should be ended with semicolan
                ((a,b)=>{
                    retutn a+b;
                })(10,20);

        Higher Order Function
            A Function which accept another Function as argument and also return a another Function is called as Higher order Function
                function multiple(a, addFun){
                    return a * addFun(2,4);
                }    

                function addFun(a,b){
                    return a+b;
                }

        Call back Function
            Function which is passed as argument for the another Function is called call back Function
                function multiple(a, addFun){
                    return a * addFun(2,4);
                }    

                function addFun(a,b){
                    return a+b;
                }

                    addFun -> is the callback Function, this will called by the another Function during execution

        Nested Function
            A Function inside the another Function is called as nested Function
                function add(a,b){
                    function inner(){
                        console.log("inner Function);
                    }

                    inner();
                }
                
        Generator Function
            It is the special type of Function, where we can stop the execution in midway and we can resume it later, where it is stoped
             by using yeild and next()
            It is represented by * along with function keyword
                function* genObj(){
                    clg("start")
                        yeild 1;
                    clg("running")
                        yeild 2;
                    clg("finished")
                }

                let gen = genObj()
                gen.next() //-> {value : 1, done : false}
                gen.nex() //-> {value : 2, done : false}
                gen.next() // -> {value : finished, done : true}

        Constructor Function

                                        this.name = name;
                                        this.age = age;
                                      }

Loops
    There are some JS special loops,
        for (of) 
            Used to iterate over values of Array and string
               let arr[] = {1,2,3,4,5}
                for(let n : arr){
                    console.log(n) // 1 2 3 4 5
                }

        for (in)  
            Used to iterate over keys/index of Array, string and object
                 for(let n : arr){
                    console.log(n) // 0 1 2 3 4
                }

        forEach()
            It is a higher order array method used to traverse through the array, which takes callback funtion as parameter,
             that callback funtion takes upto 3 arguments like element, index, array.
            We can modify the original array with the help of that callback funtion
                arr.forEach((element, index, array)=>{
                    console.log(element, index, array) // 0 1 [1,2,3,4,5] ...
                })

        .map()
            It is a higher order method used to treaverse through the array, it returns new array
            It take one callback funtion as parameter, that callback funtion can take upto 3 arguments like element, index, array        
            We can modify the original array with the help of that callback funtion
  
                arr.map((e, i, arr)=>{
                    console.log(e, i, arr); // 0 1 [1,2,3,4,5]
                })

        .filter()
            It is a higher order funtion, which is used to traverse through the array with a condition,
             it return a new array of element, which satisfied that condition            
            It take one callback funtion as parameter, that callback funtion can take upto 3 arguments like element, index, array
            We can modify the original array with the help of callback funtion

                arr.filter((e)=>{
                    return e%2==0;
                })

        .reduce()
            It is a higher order funtion, which is used convert the array into a single value
            It take one callback funtion as parameter, that callback funtion can take upto 4 arguments like accumaltor, element,index,  array
            Accumalator default value is the first element of arr,
              but we can assign the value as a 2nd parameter for reduce method
                arr.reduce((acc, e)=>{
                    return acc+e; // 15
                },0)  // 0 is the user assigned default value for Accumalator
        
        .sort()
            It is used to sort the element of array, by default it converts array element into stirng and sort them in alphabetic order,
             it returns new array

                Ass ->  arr.sort((a,b) => a - b)
                Dec ->  arr.sort((a,b) => b - a)

        .toSorted() 
            It  was introduce in 2023, which return a new array with sorted order without converting to string
                let arr1 = {5,4,3,2,1}
                arr1.toSorted() //  [1,2,3,4,5]

Promise
    A Promise is a JavaScript object that represents the eventual success or failure of an asynchronous operation.
    Syntax -> let p1 = new Promise((resolve, reject)=>{resolve("") reject("")})

    States of Promise
        1. Pending -> Promise under execution
        2. Fulfilled -> Promise Resolved (success)
        3. Rejected -> Promise Rejected (failure)

    Promise Handling Methods
        .then()      → worked as callback funtion for when promise is successfully Resolved (Fulfilled-> Handles success)
        .catch()     → Worked as callback funtionn for when promise is fails (Rejected -> Handles failure)
        .finally()   → Executes regardless of result

             p1.then((data)=>{clg(data)}).then((err)=>clg(err)).finally(clg("Handled"))
    
        .resolve() -> Used to Creates a fulfilled promise.
        .reject() -> Used Creates a rejected promise.

            let p2 = Promise.resolve("").reject("")

        .all() -> Waits for all promises to succeed.
        .allSettled() -> Waits for all promises to complete.
        .race() -> Returns the first settled/executed promise.
        .any() -> Returns the first fulfilled promise.

            Promise.all/.any/.allSettled/.race([p1,p2]).then().catch

    Uses
        API Calls, Database Operations, File Handling, Timers,Network Requests


fetch()

    A built-in JavaScript function used to send HTTP requests and communicate with APIs or servers.
        fetch() ->  returns a Promise<Response>.
    When the frontend requests data, the server usually sends the data in JSON format instead of a JavaScript object.
    To convert the JSON response into JavaScript data, we use:
        response.json() -> returns a Promise that resolves to the actual JavaScript Object or Array.

        let fetchData = fetch(url);
        fetchData.then((data)=>{
            let jsonData = data.json()
            jsonData.then((data)=>{
                data.map(e)=>{
                    clg(e)
                }
            }).catch((err)=>{clg(err)})
        }).catch((err)=>{clg(err)})

async/await

    It is a modern way to handle Promises and write asynchronous code in a synchronous-looking manner.
    We need to use try-catch inside the async funtion to handle promise rejections

    async
        It is a asynchronous funtion which always return a promise

    await
        It is a keyword inside an async function.
        It pauses the execution of the function until the Promise is resolved and returns the resolved value.

  
            let getData = async ()=>{
                try{
                    let res = await fetch(url)
                    let data = await res.json()
                    data.map((e)=>{
                        clg(e)
                    }) 
                }catch(err){
                    clg(err)
                }
            }
