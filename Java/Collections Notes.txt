List
    List is the sub-interface of Collection.
    It is used to store a collection of objects in an ordered manner.

    Advantages of List
        It allows duplicate elements.
        It maintains insertion order.
        It supports index-based access.
        It allows multiple null values.
        It supports dynamic resizing.

    Types
        ArrayList
            Uses a dynamic array internally.
            Fast for retrieving elements (get()).
            Slow for insertion and deletion in the middle.
            Not synchronized.
            Best suited for frequent read operations.

        LinkedList
            Uses a doubly linked list internally.
            Fast for insertion and deletion.
            Slow for random access (get()).
            Not synchronized.
            Can also be used as a Queue and Deque.

        Vector
            Uses a dynamic array internally.
            Synchronized (thread-safe).
            Slower than ArrayList due to synchronization.
            Legacy class.

        Stack
            Subclass of Vector.
            Follows the LIFO (Last In, First Out) principle.
            Common methods:
                push()
                pop()
                peek()

    Common Methods
        add(E e)
            Adds an element.

        add(int index, E e)
            Inserts an element at a specific index.

        get(int index)
            Returns the element at the given index.

        set(int index, E e)
            Replaces the element at the given index.

        remove(int index)
            Removes an element using its index.

        remove(Object o)
            Removes the specified object.

        contains(Object o)
            Checks whether the element is present.

        size()
            Returns the number of elements.

        isEmpty()
            Checks whether the list is empty.

        clear()
            Removes all elements.

        indexOf(Object o)
            Returns the index of the first occurrence.

        lastIndexOf(Object o)
            Returns the index of the last occurrence.

    Ways to Traverse List
        Using for loop
        Using enhanced for-each loop
        Using Iterator
        Using ListIterator
        Using forEach() method (Java 8)

    Difference Between ArrayList and LinkedList

        ArrayList
            Dynamic Array
            Fast retrieval
            Slow insertion/deletion
            Less memory usage

        LinkedList
            Doubly Linked List
            Slow retrieval
            Fast insertion/deletion
            More memory usage

    Interview Points
        List is an interface.
        It extends the Collection interface.
        It stores objects in insertion order.
        Duplicate elements are allowed.
        Null values are allowed.
        It supports index-based operations.
        ArrayList is the most commonly used implementation.

Set
    Set is the sub-interface of Collection.
    It is used to store a collection of unique objects.

    Advantages of Set
        It does not allow duplicate elements.
        It allows at most one null value (depends on implementation).
        It provides fast searching.
        It is used to store unique elements.

    Types
        HashSet
            Uses a Hash Table internally.
            Does not maintain insertion order.
            Allows one null value.
            Best for fast insertion, deletion, and searching.
            Not synchronized.

        LinkedHashSet
            Uses a Hash Table and Linked List internally.
            Maintains insertion order.
            Allows one null value.
            Slower than HashSet.

        TreeSet
            Uses a Red-Black Tree internally.
            Stores elements in sorted (ascending) order.
            Does not allow null values.
            Slower than HashSet.
            Implements the NavigableSet interface.

    Common Methods
        add(E e)
            Adds an element.

        remove(Object o)
            Removes the specified element.

        contains(Object o)
            Checks whether an element exists.

        size()
            Returns the number of elements.

        isEmpty()
            Checks whether the set is empty.

        clear()
            Removes all elements.

        iterator()
            Returns an iterator to traverse the set.

    Ways to Traverse Set
        Using enhanced for-each loop
        Using Iterator
        Using forEach() method (Java 8)

    Difference Between HashSet and LinkedHashSet

        HashSet
            Does not maintain insertion order.
            Faster.
            Uses Hash Table.

        LinkedHashSet
            Maintains insertion order.
            Slightly slower.
            Uses Hash Table + Linked List.

    Difference Between HashSet and TreeSet

        HashSet
            Unordered.
            Allows one null value.
            Faster (O(1) average).

        TreeSet
            Sorted.
            Does not allow null values.
            Slower (O(log n)).

    Interview Points
        Set is an interface.
        It extends the Collection interface.
        It stores only unique elements.
        Duplicate elements are not allowed.
        HashSet is unordered.
        LinkedHashSet maintains insertion order.
        TreeSet maintains sorted order.

Queue
    Queue is the sub-interface of Collection.
    It is used to store a collection of objects in FIFO (First In, First Out) order.

    Advantages of Queue
        It follows the FIFO principle.
        It allows duplicate elements.
        It is used for processing elements in sequence.
        It is suitable for scheduling and task processing.
        Most implementations do not support index-based access.

    Types
        PriorityQueue
            Stores elements based on priority.
            By default, elements are sorted in ascending order.
            Does not maintain insertion order.
            Does not allow null values.
            Not synchronized.

        ArrayDeque
            Uses a dynamic array internally.
            Faster than Stack for stack operations.
            Can be used as both Queue and Stack.
            Does not allow null values.
            Not synchronized.

        LinkedList
            Implements the Queue interface.
            Maintains insertion order.
            Allows null values when used as a List, but Queue operations generally discourage nulls.
            Can also be used as a Deque.

    Common Methods
        add(E e)
            Inserts an element.
            Throws an exception if insertion fails.

        offer(E e)
            Inserts an element.
            Returns false if insertion fails.

        remove()
            Removes and returns the head element.
            Throws an exception if the queue is empty.

        poll()
            Removes and returns the head element.
            Returns null if the queue is empty.

        element()
            Returns the head element.
            Throws an exception if the queue is empty.

        peek()
            Returns the head element.
            Returns null if the queue is empty.

        size()
            Returns the number of elements.

        isEmpty()
            Checks whether the queue is empty.

        clear()
            Removes all elements.

    Ways to Traverse Queue
        Using enhanced for-each loop
        Using Iterator
        Using forEach() method (Java 8)

    Difference Between add() and offer()

        add()
            Throws an exception if insertion fails.

        offer()
            Returns false if insertion fails.

    Difference Between remove() and poll()

        remove()
            Throws an exception if the queue is empty.

        poll()
            Returns null if the queue is empty.

    Difference Between element() and peek()

        element()
            Throws an exception if the queue is empty.

        peek()
            Returns null if the queue is empty.

    Interview Points
        Queue is an interface.
        It extends the Collection interface.
        It follows the FIFO (First In, First Out) principle.
        It allows duplicate elements.
        It does not support index-based access.
        PriorityQueue stores elements based on priority.
        ArrayDeque can be used as both Queue and Stack.

Map

    It is used to store data in the form of Key-Value pairs.
    Map does not extend the Collection interface.

    Advantages of Map
        Stores data as Key-Value pairs.
        Keys are unique.
        Values can be duplicated.
        Provides fast searching using keys.
        Efficient for storing and retrieving data.

    Types
        HashMap
            Uses a Hash Table internally.
            Does not maintain insertion order.
            Allows one null key.
            Allows multiple null values.
            Not synchronized.
            Best for fast retrieval.

        LinkedHashMap
            Uses a Hash Table and Linked List internally.
            Maintains insertion order.
            Allows one null key.
            Allows multiple null values.
            Not synchronized.

        TreeMap
            Uses a Red-Black Tree internally.

            Does not allow null keys.
            Allows multiple null values.
            Slower than HashMap.

        Hashtable
            Uses a Hash Table internally.
            Does not maintain insertion order.
            Does not allow null keys.
            Does not allow null values.
            Synchronized (Thread-safe).
            Legacy class.

    Common Methods
        put(K key, V value)

        get(Object key)

        values()
            Returns all values.

        entrySet()
            Returns all key-value pairs.

        size()
            Returns the number of entries.

        isEmpty()
            Checks whether the map is empty.

    Ways to Traverse Map

        Using values()
        Using entrySet()
        Using forEach() method (Java 8)

    Difference Between HashMap and LinkedHashMap

        HashMap
            Does not maintain insertion order.

            Allows one null key.

        LinkedHashMap
            Maintains insertion order.
            Slightly slower.
            Allows one null key.

    Difference Between HashMap and TreeMap

        HashMap
            Unordered.
            Allows one null key.
            O(1) average search.

        TreeMap
            Sorted by keys.
            Does not allow null keys.
            O(log n) search.

    Difference Between HashMap and Hashtable

        HashMap
            Not synchronized.
            Allows one null key.
            Allows multiple null values.
            Faster.

        Hashtable
            Synchronized.
            Does not allow null keys.
            Does not allow null values.
            Slower.

    Interview Points
        Map is an interface.

        Keys are unique.

Iterator
    Iterator is an interface present in the java.util package.
    It is used to traverse (iterate) the elements of a Collection one by one.
    It supports only forward direction traversal.

    Methods
        hasNext() ->  Returns true if the next element is available.
        next() ->  Returns the next element.
        remove() ->  Removes the last element returned by next() (Optional operation)
        forEachRemaining(Consumer action) ->  Performs the given action for all remaining elements.
                                                 Introduced in Java 8.

    Example
        ArrayList<String> list = new ArrayList<>();
        list.add("Java");
        list.add("Python");
        list.add("SQL");

        Iterator<String> itr = list.iterator();

        while (itr.hasNext()) {
            System.out.println(itr.next());  // Java Python SQL
        }

ListIterator
    ListIterator is an interface present in the java.util package.
    It is used only with List to traverses in both forward and backward directions.
    It is obtained using the listIterator() method.
    It works only with List implementations such as ArrayList, LinkedList, and Vector.

    Methods
        hasNext() -> Returns true if the next element is available.
        next() -> Returns the next element.
        hasPrevious() -> Returns true if the previous element is available.
        previous() -> Returns the previous element.
        add(E e) -> Inserts an element at the current position.
        remove() -> Removes the last element returned by next() or previous().
        set(E e) -> Replaces the last element returned by next() or previous().
        nextIndex() -> Returns the index of the next element.
        previousIndex() -> Returns the index of the previous element.

    Example

        ArrayList<String> list = new ArrayList<>();

        list.add("Java");
        list.add("Python");
        list.add("SQL");

        ListIterator<String> litr = list.listIterator();

        // Forward Traversal
        while (litr.hasNext()) {
            System.out.println(litr.next());   // Java Python SQL
        }

        // Backward Traversal
        while (litr.hasPrevious()) {
            System.out.println(litr.previous());   // SQL Python Java
        }

Comparable
    Comparable is an functional interface present in the java.lang package. 
    It contains compareTo() which helps on default sorting logic

    Method
        compareTo(T obj)
            Returns
                Negative value -> Current object is smaller than the given object.
                Zero            -> Both objects are equal.
                Positive value -> Current object is greater than the given object.

    Example

        class Student implements Comparable<Student> {

            int id;

            Student(int id) {
                this.id = id;
            }

            @Override
            public int compareTo(Student s) {
                return this.id - s.id;
            }
        }

    Sorting Order
        Ascending
            return this.id - s.id;
        Descending
            return s.id - this.id;

Comparator
    Comparator is an interface in the java.util package.
    It is used to define custom sorting logic for objects.
    The sorting logic is written outside the class.

    Method
        compareTo(T obj)
            Returns
                Negative value -> Current object is smaller than the given object.
                Zero            -> Both objects are equal.
                Positive value -> Current object is greater than the given object.

    Example
        class StudentAgeComparator implements Comparator<Student> {

            @Override
            public int compare(Student s1, Student s2) {
                return s1.age - s2.age;
            }
        }

        Collections.sort(studentList, new StudentAgeComparator());

    Sorting Order
        Ascending
            return s1.age - s2.age;

        Descending
            return s2.age - s1.age;

Functional Interface
    A Functional Interface is an interface that contains exactly one abstract method
     mainly used to support Lambda Expressions and Method References.
    It is introduced in Java 8.

    Annotation 
        @FunctionalInterface
            Ensures the interface contains only one abstract method.
            Generates a compile-time error if more than one abstract method is added.

    Rules
        Must contain exactly one abstract method.
        Can contain multiple default methods.
        Can contain multiple static methods.
        Can contain methods inherited from Object.

    Example

        @FunctionalInterface
        interface Demo {

            void display();

            default void show() {
                System.out.println("Default Method");
            }

            static void test() {
                System.out.println("Static Method");
            }
        }

Conversion (Collection Conversion)
    Conversion is the process of converting one collection type into another.
    It is used when different collection implementations are required for different operations.
    
    Common Conversions

        Array → List
            Arrays.asList(array)

        List → Array
            list.toArray()

        List → Set
            new HashSet<>(list)
            Removes duplicate elements.

        Set → List
            new ArrayList<>(set)

        List → Queue
            new LinkedList<>(list)

        Queue → List
            new ArrayList<>(queue)

        Set → Queue
            new LinkedList<>(set)

        Queue → Set
            new HashSet<>(queue)

        Map → Set
            map.entrySet()
            map.keySet()

        Map → Collection
            map.values()

    Conversion Methods

        Arrays.asList()
            Converts an array into a List.

        toArray()
            Converts a List into an Array.

        new ArrayList<>(collection)
            Converts any Collection into an ArrayList.

        new LinkedList<>(collection)
            Converts any Collection into a LinkedList.

        new HashSet<>(collection)
            Converts any Collection into a HashSet.
            Removes duplicate elements.

        new TreeSet<>(collection)
            Converts any Collection into a TreeSet.
            Sorts the elements.

    Interview Points
        Arrays.asList() converts an Array to a List.
        toArray() converts a List to an Array.
        HashSet removes duplicate elements.
        TreeSet sorts elements in ascending order.
        ArrayList maintains insertion order.
        LinkedList can be used as both a List and a Queue.
